a16z Battle of the Hacks Participant with TreeHacks

This hackathon in June 2016 was held at the Andreesen Horowitz Firm in Menlo Park, and was designed to provide the organizers of hackathons at various colleges across the U.S. an opportunity to hack. PROJECTS

degreebee

Skills: Android (Android Studio), Swift (XCode), Firebase, Google Apps Script (Google Drive API Integration)

The app aims to facilitate the college and future planning process, as it compiles useful information of majors, colleges, jobs, and references to serve as a starting point for research. The typical search process can be tedious without the centralization of information sources, when future planning should be exciting. I coded the app while my partner took care of the marketing and logo design. In addition, I partnered with a few companies in order to get the most relevant data in our app and I wrote a script in order to add their large amounts of data into the Firebase database, which dynamically feeds data into the app. I also included the option for users to save the information to a Google Drive file, as I wanted the app to directly allow the user to compile a list of what they are interested in for further research. This app made it to the semifinals of the Technovation challenge and the website walking through the app's features can be found here. It is on Android and iTunes in order to make it as accessible as possible!

Pic-to-Word

Skills: Swift (XCode), Google Apps Script (Google Drive API Integration), Google Tesseract OCR (Character Recognition), Matlab (Machine Learning)

This app utilizes Optical Character Recognition to convert text from images to digital text, allowing the user to further process that text and upload it to Google Drive as a document. It was something I really needed in school and I thought it would bring much-needed convenience others. In order to more deeply understand the algorithm, as I was using the Tesseract API, I took Andrew Ng's Machine Learning Course on Coursera, learning various methods and how to use Matlab. Currently, the Tesseract does not produce very reliable results, even with many different image processing techniques. It is most accurate for typed text, such as in books and newspaper. However, I hope that one day I can write my own machine learning algorithm in order to produce accurate results for handwritten text.

Pocket Check

Skills: Swift (XCode), Google Tesseract OCR (Character Recognition)

This app is a financial budgeting app that keeps track of receipts and total spending amounts by spending categories. It was developed during the 4 week Girls Who Code Summer Immersion Program at VMWare with a team of 4 others. I served as the main programmer, as I was really determined to learn Swift and XCode. Users can add entries, and we even included Tesseract OCR in an attempt to make it more convenient for users to enter the necessary information. The app also includes visuals, breaking down the spendings by categories, which are denoted by nice-looking pastel colors. There is also the option to set a budget and a calendar to see which days users have spent money on. User accounts and data were all stored in Parse database. It can be found on Github here.

Learn Music Theory

Skills: Java

This game is meant to help students remember the sharps and flats in all major and minor scales in a fun little game. It was started in my 9th grade Java class and later improved over the summer. The player controls this panda to collect food items, shields, coins, while avoiding the trash items. The user has to collect a certain amount of points before advancing to the next level. Every 15 seconds, the game will be paused and a music theory question will be asked in a multiple choice format. The player has to get it right before they can continue with the game. If they get 5 wrong, the game is over and they can review the wrong questions. The coins can be used to purchase new cute characters from the store. I hope to develop it as an app to help children learn basic music theory, as I myself have lots of trouble memorizing the sharps and flats of scales. The code for the game can be found on Github here (although it is very disorganized as I hadn't used git before) and it can played by downloading the jar file here.
